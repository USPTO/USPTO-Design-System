Design Guidance for using Section 508 Requirements

Prerequisites

Before utilizing Section 508 guidance, ensure you are familar with the purpose amd scope of standards developed by the World Wide Web Consortium (W3C), and Web Content Accessibility Guidelines (WCAG), includings WCAG 2.0 and WCAG 2.1.

Introduction

This document is to be used by developers who create user interface functionality, and need to understand the applicability of and the specific requirements for compliance with Web Content Accessibility Guideline (WCAG). The WCAG requirements have a long history of development and is still evolving. The section below provides the steps to use for ensuring compliance in the development of user interface components.

How to Use

  • Start with the UI element you plan to use in your design.
  • Review the Level A and AA compliance chart
  • For specific detaIls regarding how to meet he applicable WCAG success criteria for the UI element, you may refer to the list provided below.
  • Implement the design and coding per the success criteria and the guidelines in the Component Section of the USPTO Design System.
